خصائص الإرسال الفيزيائي

Caractéristiques d'une transmission physique · Physical Transmission Characteristics

01 · BANDWIDTH

عرض النطاق الترددي Bande Passante Bandwidth

الخاصية الأساسية لوسيط الإرسال. يتصرف الوسيط كمرشح يسمح فقط لنطاق محدود من الترددات بالمرور. كل تردد خارج هذا النطاق يُضعَّف بشدة.

La largeur de la bande passante est la caractéristique essentielle d'un support de transmission, qui se comporte comme un filtre.

أمثلة / Exemples:
🎙️ خط هاتفي: 300 Hz → 3400 Hz (3100 Hz)
📡 كابل محوري: 300 → 400 MHz
👂 الأذن البشرية: 20 Hz → 20 kHz
FORMULA
Bandwidth = F_max - F_min
02 · MODULATION RATE

سرعة التعديل Rapidité de Modulation Modulation Rate / Baud Rate

تُعبِّر عن عدد الرموز (symboles) المُرسَلة في وحدة الزمن. وحدتها: الباود (Baud). إذا كان T هو مدة الفترة بين قيمتين متتاليتين:

FORMULAS
R = 1 / T    (bauds)
R_max = 2 × F_max    (Nyquist)
⚠️ الباود ≠ bit/s — الباود هو عدد الرموز، وليس البتات بالضرورة
03 · BIT RATE

الدبيت / تدفق البيانات Débit Binaire Bit Rate

عدد البتات المُرسَلة في وحدة الزمن. وحدته: bit/s. يرتبط بسرعة التعديل وعدد حالات الإشارة (valence V).

FORMULAS
D = R × log₂(V)
D_max = 2F × log₂(V)    (Nyquist)
C = F × log₂(1 + S/N)    (Shannon)
04 · ATTENUATION

التوهين / الضعف Affaiblissement / Atténuation Attenuation

قناة الإرسال تُضعِّف الإشارة أثناء انتشارها. يزداد التضعيف مع المسافة. يُقاس بنسبة القدرة عند الخروج إلى القدرة عند الدخول.

FORMULAS
A = S_out / S_in
A (dB) = 10 × log₁₀(S/N)    décibels
💡 S = القدرة عند الخروج، N = القدرة عند الدخول
05 · ERROR RATE

معدل الأخطاء Taux d'Erreur Bit Error Rate (BER)

احتمال فقدان أو تشويه معلومة (bit واحد). يُحسب بقسمة عدد البتات الخاطئة على إجمالي البتات المُرسَلة خلال مدة كافية.

FORMULA
TEB = Bits_erronés / Bits_émis
مثال: 1 خطأ من أصل مليون bit → TEB = 10⁻⁶
06 · THEOREMS

نظريتا نيكويست وشانون Théorèmes de Nyquist & Shannon Nyquist & Shannon Theorems

نيكويست (خط بدون ضوضاء): الدبيت الأقصى على خط بلا ضوضاء يساوي ضعف عرض النطاق مضروباً في logarithme ثنائي للـ valence.

شانون (خط مع ضوضاء): السعة القصوى تعتمد على نسبة الإشارة للضوضاء S/N.

KEY FORMULAS
Nyquist:D_max = 2F × log₂(V)
Shannon:C = F × log₂(1 + S/N)
V = valence:2 (bivalent), 4 (quadrivalent), 16...
07 · PROPAGATION

زمن الانتشار Temps de Propagation & Transmission Propagation & Transmission Time

وقت إرسال الإطار يتكون من وقتين: زمن الإرسال (وقت وضع البتات على الوسيط) + زمن الانتشار (وقت وصول الإشارة).

FORMULAS
T_trans =Taille_trame / Débit
T_prop =Distance / Vitesse_prop
T_total =T_trans + T_prop
08 · RING NETWORK

الشبكة الحلقية Réseau en Anneau (Token Ring) Ring Network

شبكة حلقية تتضمن محطات موزعة على الحلقة. تُحسب: عدد البتات في الحلقة، عدد الإطارات في العبور، والمردود.

FORMULAS
T_bit =1 / Débit
Bits_prop =T_prop_anneau × Débit
Rendement =T_trame / (T_trame + T_prop)

التمارين المحلولة

TD 1 — Exercices avec solutions complètes

01
السؤال: ما هي سرعة التعديل R اللازمة لتحقيق دبيت D = 2400 bit/s، إذا كانت الإشارات ثنائية القيمة (bivalent, V=2)؟ وما هو الحال إذا كانت رباعية (V=4)؟
📐 القانون المستعمل: R = D / log₂(V)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 الحالة 1 — V = 2 (bivalent)
R = 2400 / log₂(2) = 2400 / 1
→ R = 2400 bauds
// الإشارة بحالتين فقط (0 و 1): كل رمز = 1 bit
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 الحالة 2 — V = 4 (quadrivalent)
R = 2400 / log₂(4) = 2400 / 2
→ R = 1200 bauds
// الإشارة بـ 4 حالات: كل رمز = 2 bits — نصف السرعة كافية!
02
السؤال: خط هاتفي بترددات 300-3400 Hz. سرعة التعديل R = 1200 bauds، والـ valence V = 16. ما هو الدبيت D؟
📐 القانون المستعمل: D = R × log₂(V)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
D = 1200 × log₂(16)
D = 1200 × 4    // log₂(16) = log₂(2⁴) = 4
→ D = 4800 bit/s
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// V=16 تعني 4 bits per symbol → الدبيت أعلى من سرعة التعديل
03
السؤال: نظام إرسال رقمي بدبيت 9600 bit/s. إذا كان كل رمز يُشفر كلمة من 4 bits، ما هي أدنى بنده passante لازمة؟
📐 تحديد المعطيات:
D = 9600 bit/s، كل رمز = 4 bits → V = 2⁴ = 16
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 الخطوة 1: حساب R
R = D / log₂(V) = 9600 / log₂(16) = 9600 / 4 = 2400 bauds
🔹 الخطوة 2: R_max = 2 × F_max → F_min = R / 2
F_min = R / 2 = 2400 / 2
→ F_min = 1200 Hz
// الحد الأدنى من البنده passante هو 1200 Hz
04
السؤال: احسب الدبيت الأقصى لزوج ملتوي Cat5 (بنده passante = 100 MHz)، إذا كانت قدرة الإشارة تساوي 100 ضعف قدرة الضوضاء.
📐 المعطيات: F = 100 MHz = 10⁸ Hz، S/N = 100
📐 القانون: C = F × log₂(1 + S/N) — Shannon
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
C = 10⁸ × log₂(1 + 100)
C = 10⁸ × log₂(101)
C = 10⁸ × (log₁₀(101) / log₁₀(2))
C = 10⁸ × (2.0043 / 0.30103)
C = 10⁸ × 6.658
→ C ≈ 665.8 Mbit/s
05
السؤال: ما هو الدبيت الأقصى في قناة بعرض 3 kHz ونسبة إشارة/ضوضاء = 20 dB؟
📐 تحويل dB أولاً:
S/N = 10^(20/10) = 10^2 = 100
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
C = F × log₂(1 + S/N)
C = 3000 × log₂(1 + 100) = 3000 × log₂(101)
C = 3000 × 6.658
→ C ≈ 19 974 bit/s ≈ 20 kbit/s
// ⚠️ انتبه: دائماً حوّل dB إلى نسبة قبل Shannon!
06
السؤال: ألياف بصرية: D = 155 Mb/s، L = 3000 km، V = 2×10⁸ m/s. كم من الوقت لتلقي نهاية حزمة 512 octet؟ ثم قارن مع زوج ملتوي D = 2 Mb/s.
📐 الألياف البصرية:
T_emission = (512 × 8) / (155 × 10⁶) = 4096 / 155×10⁶
T_em = 2.643 × 10⁻⁵ s = 26.43 μs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
T_propagation = 3000×10³ / (2×10⁸) = 3×10⁶ / 2×10⁸
T_prop = 0.015 s = 15 ms
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
T_total = 15 ms + 26.43 μs ≈ 15.026 ms
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📐 الزوج الملتوي (D = 2 Mb/s):
T_emission = 4096 / (2×10⁶) = 2.048 ms
T_propagation = 15 ms (نفس المسافة)
T_total = 15 ms + 2.048 ms = 17.048 ms
// الفرق صغير لأن زمن الانتشار يهيمن (المسافة كبيرة)
07
السؤال: شبكة حلقية: 10 محطات، V_prop = 200 m/μs، الإطار = 256 bit.
a) L=10km, D=5Mbit/s   b) L=1km, D=500Mbit/s   c) مقارنة المردود
📐 المعطيات المشتركة:
T_trame = 256 / D
T_prop_anneau = L / V_prop = L / (200×10³ m/s)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 a) L=10km, D=5Mbit/s
T_bit = 1/(5×10⁶) = 0.2 μs
T_prop = 10000 / (200×10⁶) = 50 μs
Bits_prop = 50μs × 5Mbit/s = 250 bits en propagation
T_trame = 256/(5×10⁶) = 51.2 μs
T_total_AR = T_trame + 2×T_prop = 51.2 + 100 = 151.2 μs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 b) L=1km, D=500Mbit/s
T_bit = 1/(500×10⁶) = 2 ns
T_prop = 1000 / (200×10⁶) = 5 μs
Bits_prop = 5μs × 500Mbit/s = 2500 bits en propagation
T_trame = 256/(500×10⁶) = 0.512 μs
T_total_AR = 0.512 + 10 = 10.512 μs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔹 c) المقارنة والمردود
η_a = T_trame / T_total = 51.2 / 151.2 = 33.9%
η_b = T_trame / T_total = 0.512 / 10.512 = 4.87%
// الحلقة b أسرع لكن مردودها أقل بكثير!

أنواع الترميز والتشفير

Codages numériques · Digital Encoding Methods

BASEBAND · UNIPOLAR

الترميز الأحادي القطب Codage Unipolaire · Unipolar Encoding

1 مستوى جهد موجب V (الجهد العالي)
0 مستوى جهد صفر (0V)
مثال: 0110010
0=0V, 1=V, 1=V, 0=0V, 0=0V, 1=V, 0=0V
BASEBAND · NRZ

الترميز NRZ Non Return to Zero

1 جهد موجب +V طوال مدة البت
0 جهد سالب -V طوال مدة البت
لا يوجد عودة إلى الصفر — لكن صعب مزامنة
BASEBAND · NRZI

الترميز NRZI Non Return to Zero Inverted

1 تحويل (transition) عند بداية البت — يعكس ما كان قبله
0 بقاء على نفس مستوى الجهد — يعامل كما كان قبله
BASEBAND · MANCHESTER

ترميز مانشستر Codage Manchester

1 من +V إلى -V في منتصف البت (هبوط)
0 من -V إلى +V في منتصف البت (صعود)
✅ ميزة: المزامنة مضمونة — دائماً تغيير في المنتصف
BASEBAND · MANCHESTER DIFF

مانشستر التفاضلي Manchester Différentiel

1 يعكس عند بداية البت ما كان قبله، مع transition في المنتصف
0 يبقى كما كان في البداية، مع transition في المنتصف
BASEBAND · MILLER

ترميز ميلر Codage Miller / Modified FM

1 تحويل في منتصف البت (مثل Manchester)
0 إذا تبعه 0 آخر: تحويل في نهاية البت. وإلا: لا تحويل
نهاية الـ 1 = بداية الـ 1 التالي
BROADBAND · MODULATION

التعديل الترددي — FM Modulation de Fréquence / FSK

1 تردد عالٍ (fréquence haute)
0 تردد منخفض (fréquence basse)
BROADBAND · MODULATION

التعديل الطوري — PM Modulation de Phase / PSK

1 من الفوق للتحت (inversion de phase)
0 من التحت للفوق
BROADBAND · MODULATION

التعديل السعي — AM Modulation d'Amplitude / ASK

1 سعة عالية (amplitude haute)
0 سعة صفر أو منخفضة جداً

كشف وتصحيح الأخطاء

Détection et Correction d'Erreurs · Error Detection & Correction

VRC / PARITÉ

بت التحقق (المساواة) VRC — Contrôle de Parité Vertical Vertical Redundancy Check

يُضاف bit واحد لكل كلمة بحيث يكون مجموع الـ 1 في الكلمة زوجياً (parité paire) أو فردياً (parité impaire).

RULES
Paire:nb de 1 → pair → bit = 0
Paire:nb de 1 → impair → bit = 1
Impaire:inverse de paire
مثال "I" ASCII = 1001001 (3 بتات 1 → فردي)
VRC pair → أضف 1 → 10010011
VRC impair → أضف 0 → 10010010
LRC / PARITÉ HORIZONTALE

تحقق خطي طولي LRC — Longitudinal Redundancy Check Horizontal Parity

يُحسب بit التحقق لكل عمود (موضع bit) عبر جميع الكلمات. يُضاف كسطر إضافي في النهاية.

للكشف عن الأخطاء المنفردة والمتعددة في نفس العمود.
يستعمل مع VRC للحصول على حماية أفضل.
CRC / POLYNOMIAL

التحقق الدوري بالكثيرة الحدود CRC — Cyclic Redundancy Check CRC Polynomial Code

يُعدّ الأقوى للكشف عن الأخطاء. يقوم على قسمة رسالة M(x) على كثيرة حدود G(x) واستعمال الباقي R(x) كـ checksum.

STEPS
1. M(x).x^nأضف n صفر لـ M (n = درجة G)
2. قسّمM(x).x^n ÷ G(x)
3. R(x)الباقي → هو الـ CRC
4. T(x)M(x).x^n - R(x) = الرسالة النهائية
⚠️ في الحساب الثنائي: XOR بدل الطرح!
HAMMING CODE

رمز هامينغ — تصحيح الأخطاء Code de Hamming Hamming Error-Correcting Code

رمز يكشف ويُصحّح خطأ بت واحد. يضيف بتات تحقق P في مواضع قوى 2.

PARITY BIT POSITIONS
P₁ (pos 1):يغطي: 1,3,5,7,9,11...
P₂ (pos 2):يغطي: 2,3,6,7,10,11...
P₄ (pos 4):يغطي: 4,5,6,7,12,13,14...
P₈ (pos 8):يغطي: 8-15, 24-31...
القاعدة: Pₖ يغطي k موضع، يتخطى k، يغطي k، إلخ
عدد بتات البيانات m → عدد بتات التحقق r بحيث: 2^r ≥ m + r + 1

📖 مثال CRC من الملخص

مثال: الرسالة = 1011011 ، G(x) = x⁴+x+1 = 10011

الخطوات:

M(x) = 1011011
n = 4 (درجة G) → أضف 4 أصفار
M(x).x⁴ = 10110110000
القسمة: 10110110000 ÷ 10011
الباقي R(x) = 1100
T(x) = 10110111100

ملاحظات:

في XOR: 1⊕1=0, 0⊕0=0, 1⊕0=1
G(x) يجب أن يبدأ وينتهي بـ 1
درجة الباقي يجب أن تكون أقل من درجة G(x)
في الاستقبال: T(x) ÷ G(x) → إذا الباقي = 0 ✅ لا خطأ
